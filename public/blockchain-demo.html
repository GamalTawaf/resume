What is a Blockchain?

1. Blocks & Data

A blockchain is a chain of blocks, each holding some data. Below, the first block (the "genesis block") is the root everything else builds on.

2. Hashing

Every block gets a fingerprint — a SHA-256 hash — computed from its contents. Change one character of the input and the hash comes out completely different. Try it:

(empty)

3. Linking Blocks

Each block stores the previous block's hash as its "prevHash." That link is what turns a list of blocks into a chain — each one points back at the one before it.

4. Proof of Work

Adding a block isn't free: the app must search for a "nonce" value that makes the block's hash start with N zero hex digits (each hex digit = 4 bits). Higher difficulty means more attempts — cheap to verify, expensive to produce.

5. Immutability

Try it: edit any block's data below and watch it — and everything after it — turn red. That's the whole point: once data is sealed into a hash, tampering is instantly visible.
